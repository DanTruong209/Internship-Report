# AWS Internship Worklog

**On this page**, I introduce my internship journey, including the learning process, practical AWS experiments, architecture design, and implementation of the IRMS (Incident Response Management System) project.

The internship was completed over **12 weeks**, starting from learning AWS fundamentals to designing and developing a real-world Serverless Security Management System on AWS.

During the first phase (**Week 1 - Week 8**), I focused on building a strong AWS foundation:
- Understanding Cloud Computing concepts and AWS Global Infrastructure.
- Learning core AWS services such as EC2, S3, RDS, VPC, IAM, Lambda, and Security services.
- Practicing AWS Console operations and basic deployment workflows.
- Studying cloud architecture concepts including High Availability, Microservices, CI/CD, and Event-Driven Architecture.

During the second phase (**Week 9 - Week 12**), I applied the learned AWS knowledge to design and develop the IRMS project:
- Designing the overall Serverless AWS architecture.
- Building security monitoring and incident response workflows.
- Integrating AWS services such as Lambda, API Gateway, DynamoDB, S3, EventBridge, SNS, GuardDuty, and Amazon Bedrock/Groq AI.
- Developing AI Assistant features for security incident analysis.
- Testing, monitoring, documenting, and finalizing the system.
